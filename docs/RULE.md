# RULE.md — 절대 규칙 및 예외 처리 기준

> IQ Spark Studio의 모든 작업에 적용되는 절대 규칙입니다.
> 규칙 위반 시 즉시 중단하고 대표님에게 보고합니다.

## 🛑 운영 규칙 (RULE 0~18)

### 최우선 규칙

#### RULE 0 — 사전 보고
- 모든 게시/API 호출은 **대표님 승인 후** 실행
- 승인 없이 외부 배포 절대 금지

#### RULE 1 — Zero Hallucination
- 존재하지 않는 수치/통계 생성 절대 금지
- 모든 데이터는 출처 명시 필수
- 확인 불가능한 정보는 "확인 필요"로 표기

#### RULE 2 — 수학적 무결성
- 정답 **역산 3중 검증** 필수
- 검증 실패 시 즉시 중단 + 대표님 보고
- 3중 검증 방법:
  1. 순방향 풀이 (문제 → 정답)
  2. 역방향 검증 (정답 → 문제 조건 부합 확인)
  3. 대안 풀이 (다른 방법으로 동일 정답 도출)

### 콘텐츠 생산 규칙

#### RULE 3 — 브랜드 템플릿
- 색상 팔레트:
  - 다크 배경: `#0F0F1A`
  - 앰버 골드: `#F59E0B`
  - 인디고 글로우: `#6366F1`
- 폰트: **Inter**
- 문제 원본 **1:1 클론**, 임의 변형 금지
- 해상도: **1080×1920** (9:16 세로형)

#### RULE 4 — URL 직접 노출 금지
- TikTok/IG/Shorts/Threads 캡션에 URL 직접 노출 **절대 금지**
- 허용 CTA: `"Link in bio!"` 만 사용
- **예외**: YouTube 롱폼 설명란에만 `iqspark.digital` 링크 허용

#### RULE 5 — US English 필수
- 모든 영상 텍스트, 캡션, 해시태그는 **미국 영어**로 작성
- 한국어는 별도 요청 시에만 사용
- Mensa/Raven/WAIS 수준의 논리 퍼즐 난이도 유지

#### RULE 6 — 하이브리드 정답 전략
- Hook 프레임 없이 문제 즉시 시작
- **P1 정답**: 영상에서 직접 공개 (Reveal Mode)
- **P2 정답**: 고정 댓글에서만 공개 (Comment Mode)
- 댓글 유도: `"💬 Comment your answer 👇"`
- 사이트 유입: `"Check the full solution at iqspark.digital"`

#### RULE 7 — 난이도 순서
- **Easy(P1) → Hard(P2)** 고정 순서
- 순서 변경 금지

### 배포 규칙

#### RULE 8 — AI 고지
- **TikTok**: 내장 라벨 필수 사용
- **기타 플랫폼**: `#AIGenerated` 해시태그 권장

#### RULE 9 — 워터마크 금지
- 타 플랫폼 워터마크가 포함된 영상을 다른 플랫폼에 게시 금지
- 각 플랫폼용 별도 영상 렌더링

#### 🔴 RULE 10 — 6프레임 타이밍 규격 (60초 고정)

- 콘텐츠는 반드시 아래 **6프레임 타이밍 구조**를 정확히 만족하여 **총 60초** 영상으로 생성되어야 한다.
- **구조 및 세부 타이밍**:
  - `Frame 1: Intro` → 훅(Hook) 문구 및 도전의식 자극 (~2초)
  - `Frame 2: Question 1` → Easy 문제 및 선택지 풀이, 카운트다운 타이머 24초
  - `Frame 3: Answer 1` → Question 1 정답 및 해설 공개 (6초 — Question 1의 24초 풀이 시간과 합쳐 총 30초 사이클)
  - `Frame 4: Question 2` → Hard 문제 및 선택지 풀이, 카운트다운 타이머 24초
  - `Frame 5: Comment CTA` → Question 2 정답 미공개 및 댓글 유도 (~2초)
  - `Frame 6: Final CTA` → iqspark.digital 사이트 유입 유도 카드 (~2초)
- **합산**: 정확히 `60초` (Shorts 최적 규격)

## 🔴 RULE 11 — 음악 및 오프라인 페이드 자동화

- 영상 렌더링 또는 화면 녹화 시 사용되는 BGM은 반드시 **60초 크롭** 및 **앞뒤 2초 페이드** 처리가 자동으로 입혀져야 한다.
- **볼륨**: 전체 사양의 **35% 고정**
- **페이드**: 시작 시 **1초 페이드인**, 종료 시 **2초 페이드아웃** 오프라인 렌더링 게인 곡선 적용 (`audio-clip.js` 준수)
- BGM 음원은 대표님이 직접 업로드한 로컬 음원만 사용하며, 임의의 상업 음원 무단 사용을 절대 금지한다.

## 🔴 RULE 12 — 영상 출력 및 폴더 저장 규격

- 웹앱 상의 **`Render MP4`** 버튼 클릭 시, 브라우저 단에서 `ffmpeg.wasm`을 활용하여 6프레임 이미지와 오디오 버퍼를 병합한 **MP4 동영상 파일(9:16 portrait 및 16:9 landscape)을 직접 생성하고 즉시 다운로드**를 트리거한다.
- 개별 이미지로의 활용을 위한 **`Save ALL 6 Frames`** 기능을 동시에 지원하며, 클릭 시 브라우저 다운로드 폴더 내에 `IQSpark_Desktop/<날짜>/iqspark_<날짜>_frame_<1~6>.png` 경로 형태로 압축 패키징(ZIP)되어 저장되어야 한다.

#### RULE 13 — 성과 기반 최적화
- 게시 48시간 후 데이터 수집
- 상위/하위 20% 패턴 분석
- 분석 결과를 다음 콘텐츠에 반영

#### RULE 14 — 계정 고정
- 반드시 **iqspark.info@gmail.com** 계정 사용
- 다른 계정으로의 게시 금지

#### RULE 15 — 업로드 방식
- 시스템은 직접 업로드하지 않음
- `메타데이터_복사용.txt` 생성 → **대표님이 직접 복사·붙여넣기 업로드**

### 파일 관리 규칙

#### RULE 16 — 입력 폴더 규칙
- 입력 파일: `reports/YYYYMMDD/input/`
  - `p1.png` — Easy 문제
  - `p2.png` — Hard 문제
  - `bgm.mp3` — 배경음악 (선택)
- 원본 파일 **읽기 전용**, 수정 금지

#### RULE 17 — BGM 정책
- 대표님이 제공한 경우에만 BGM 적용
- 볼륨: **35%**
- Fade In: **1초**
- Fade Out: **2초**
- 제공되지 않으면 무음 처리

#### RULE 18 — Hook 골든포맷
- **10단어 이내**
- **숫자 포함** 필수
- **0.5초 내 인지 가능**해야 함
- 예시: `"Only 2% get BOTH right 🧠"`

## 🌐 웹앱 전용 규칙 (Genspark IQ Spark Studio)

> Genspark 에이전트 페이지에서 확인된 웹앱 동작 관련 규칙

### WEBAPP RULE 1 — 중복 출제 방지
- `localStorage`에 문제 지문 서명을 기록
- 중복 검출 시 시드 재시도 (최대 200회)

### WEBAPP RULE 2 — 유입 전략 고수
- Question 1: Reveal Mode (정답 공개형)
- Question 2: Comment Mode (댓글 유도형)

### WEBAPP RULE 3~4 — 사용자 업로드 이미지
- 사용자가 퍼즐 이미지를 업로드할 경우 자동 생성 문제 대신 해당 슬롯을 **완전 대체**

### WEBAPP RULE 5 — 템플릿 재구성
- 업로드된 이미지는 날것으로 쓰지 않음
- 항상 IQ Spark 템플릿(딥블루 + 골드/시안 테두리) 내부에 맞춤형으로 재구성

### WEBAPP RULE 6 — html2canvas 버그 방지
- 훅 텍스트 투명도 버그 방지
- 불투명 실색(흰색) + Cyan 글로우 렌더링 유지

### WEBAPP RULE 7~8 — 타겟 & 난이도
- 타겟: US Audience
- 모든 카드 출력 문구: 영문(US)
- 난이도: Mensa/Raven/WAIS 논리 퍼즐 수준

### WEBAPP RULE 11 — Web Audio API
- 로컬 Web Audio API 활용
- 음원 시작 지점 설정 시 정확히 **60초 재생**
- 앞뒤 **2초 페이드 인/아웃** 실시간 오토메이션 적용

### WEBAPP RULE 12 — 다운로드 자동 분류
- 다운로드 시 브라우저 다운로드 폴더 내 `IQSpark_Desktop/<날짜>/` 디렉토리 형태로 자동 분류

## ⚠️ 예외 처리 기준

### 예외 허용 조건
1. **대표님의 명시적 서면/구두 승인**이 있는 경우
2. 승인 일시, 내용, 범위를 CONTEXT.md에 기록

### 예외 불허 항목 (어떤 상황에서도 예외 없음)
- RULE 0 (사전 보고)
- RULE 1 (Zero Hallucination)
- RULE 2 (수학적 무결성)
- RULE 14 (계정 고정)

### 예외 승인 프로세스

```
1. 에이전트가 예외 필요성 보고
2. 대표님 검토 및 승인/거부
3. 승인 시: CONTEXT.md에 예외 사항 기록
4. 해당 작업에만 일회성 적용 (영구 변경은 RULE.md 수정 필요)
```

## 📋 규칙 위반 시 처리 절차

1. **즉시 중단**: 해당 작업 진행 중단
2. **보고**: 에스컬레이션 보고 형식에 따라 대표님에게 보고
3. **원인 분석**: 위반 원인 파악 및 기록
4. **재발 방지**: 필요 시 프로세스/체크리스트 보완
